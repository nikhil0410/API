# Moving This Project to Another Machine

How to relocate the Student Success pipeline to a new computer and get it
running again.

## What to copy vs. leave behind

**Copy (the source):**

- The whole `student-success-pipeline/` folder — but **exclude** `.venv/`,
  `__pycache__/`, and `outputs/`.
- **`data.csv`** — the code loads it via `PROJECT_ROOT.parent / "data.csv"`
  (see [config.py](config.py)), so `data.csv` must sit **one level above** the
  project folder, exactly as it does now (inside the `Assignment/` folder).
  Easiest: copy the entire `Assignment/` folder so the layout is preserved.

**Do NOT copy:**

- `.venv/` — it hardcodes absolute paths to the current machine; recreate it.
- `outputs/mlflow.db` and `outputs/mlartifacts/` — MLflow stores **absolute**
  artifact paths, so they break on a new path. Re-run to regenerate.
- `__pycache__/`, `.DS_Store`.

The `.gitignore` in this folder enforces all of the above automatically.

## Method A — via Git (recommended)

On the current machine:

```bash
cd "student-success-pipeline"
git init && git add . && git commit -m "Student success pipeline"
# push to GitHub, or just copy the folder
```

Then also move `data.csv` (it lives outside the project and is excluded by
location).

## Method B — zip / copy

```bash
cd "/path/to/API"
zip -r assignment.zip Assignment \
  -x "*/.venv/*" "*/__pycache__/*" "*/outputs/*" "*/.DS_Store"
```

This bundles `data.csv` + the project while skipping the machine-specific parts.

## Setup on the new machine

```bash
cd student-success-pipeline
python3 -m venv .venv          # use Python 3.12
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

**macOS + XGBoost caveat:** XGBoost needs the OpenMP runtime, or
`import xgboost` fails:

```bash
brew install libomp
```

(Not needed on most Linux; Windows wheels bundle it.)

Then run:

```bash
python data_pipeline.py       # regenerates outputs/
python ml_pipeline.py         # retrains + repopulates MLflow (sqlite)
python flows.py               # full Prefect flow
mlflow ui --backend-store-uri sqlite:///outputs/mlflow.db   # from project root
```

## Portability notes

- All paths in [config.py](config.py) are derived from `__file__`, so the
  project runs from any location **as long as `data.csv` stays one level up**.
  To make it self-contained, move `data.csv` into the project folder and change
  `RAW_DATA_PATH` in config.py to `PROJECT_ROOT / "data.csv"`.
- Use the same Python minor version (3.12) for the pinned dependencies. Since
  `outputs/` is regenerated by re-running, you never need to move the trained
  model or the MLflow store between machines.
